### Phase 1: Root Cause Investigation

#### 第一轮错误：Repository层JPA依赖（10个错误）

**错误信息：**
```
错误: 程序包org.springframework.data.jpa.repository不存在
错误: 找不到符号: 类 JpaRepository
错误: 找不到符号: 类 Page
错误: 找不到符号: 类 Pageable
```

**根因分析：**
1. ✅ 项目已从JPA迁移到MyBatis-Plus
2. ✅ Service层已正确使用Mapper接口
3. ✅ Repository接口是旧的JPA代码，应该删除
4. ✅ 但Repository文件在文件系统中仍然存在

**关键发现：**
- 通过文件系统检查确认：Repository目录中仍有3个文件
- 编译器仍在编译这些JPA接口
- 这是架构迁移不彻底导致的

#### 第二轮错误：MyBatis-Plus分页插件（1个错误）

**错误信息：**
```
错误: 找不到符号
import com.baomidou.mybatisplus.extension.plugins.inner.PaginationInnerInterceptor;
  符号:   类 PaginationInnerInterceptor
  位置: 程序包 com.baomidou.mybatisplus.extension.plugins.inner
```

**根因分析：**
1. ✅ MyBatis-Plus版本：3.5.9
2. ✅ 检查依赖树：`mybatis-plus-spring-boot3-starter:3.5.9` 已下载
3. ✅ **查阅官方文档**（关键步骤）：发现重要变更

**官方文档说明：**
> **于 v3.5.9 起，PaginationInnerInterceptor 已分离出来。如需使用，则需单独引入 mybatis-plus-jsqlparser 依赖**

**根因确认：**
- MyBatis-Plus 3.5.9版本将分页插件分离到独立依赖
- 项目缺少 `mybatis-plus-jsqlparser` 依赖
- 这是依赖版本兼容性问题

#### 第三轮错误：BusinessException参数类型（2个错误）

**错误信息：**
```
错误: 不兼容的类型: String无法转换为Throwable
throw new BusinessException("AI_SCORE_ERROR", "AI评分失败: " + e.getMessage());
throw new BusinessException("PARSE_ERROR", "解析评分结果失败");
```

**根因分析：**
1. ✅ 检查BusinessException构造函数：
   ```java
   BusinessException(String message)
   BusinessException(String message, Throwable cause)
   ```
2. ✅ AIService错误使用：传入两个String参数
3. ✅ Pattern Analysis：其他Service都是单参数用法
4. ✅ 根因：参数类型不匹配

### Phase 4: Implementation

#### 修复1：删除JPA Repository文件

**删除的文件：**
```
src/main/java/com/jinmo/aiwriting/repository/
├── ApiConfigRepository.java
├── EssayRepository.java
└── EssayScoreRepository.java
```

**原因：**
- 项目已迁移到MyBatis-Plus
- Mapper接口已存在并正确使用
- Repository接口是遗留代码

**替代实现：**
```
src/main/java/com/jinmo/aiwriting/mapper/
├── ApiConfigMapper.java      (extends BaseMapper<ApiConfig>)
├── EssayMapper.java           (extends BaseMapper<Essay>)
└── EssayScoreMapper.java      (extends BaseMapper<EssayScore>)
```

#### 修复2：添加MyBatis-Plus分页插件依赖

**文件：** `build.gradle`

**修改前：**
```gradle
// MyBatis-Plus
implementation 'com.baomidou:mybatis-plus-spring-boot3-starter:3.5.9'
```

**修改后：**
```gradle
// MyBatis-Plus
implementation 'com.baomidou:mybatis-plus-spring-boot3-starter:3.5.9'
// 分页插件依赖（3.5.9版本需要单独引入）
implementation 'com.baomidou:mybatis-plus-jsqlparser:3.5.9'
```

**文件：** `src/main/java/com/jinmo/aiwriting/config/MybatisPlusConfig.java`

**修改后：**
```java
/**
 * 分页插件
 * 注意：MyBatis-Plus 3.5.9版本需要单独引入mybatis-plus-jsqlparser依赖
 */
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    // 添加分页插件，指定数据库类型为PostgreSQL
    interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.POSTGRE_SQL));
    return interceptor;
}
```

#### 修复3：修正AIService异常处理

**文件：** `src/main/java/com/jinmo/aiwriting/ai/AIService.java`

**修改前（错误）：**
```java
// Line 51
throw new BusinessException("AI_SCORE_ERROR", "AI评分失败: " + e.getMessage());

// Line 69
throw new BusinessException("PARSE_ERROR", "解析评分结果失败");
```

**修改后（正确）：**
```java
// Line 51 - 传入异常对象作为cause
throw new BusinessException("AI评分失败: " + e.getMessage(), e);

// Line 69 - 传入异常对象作为cause
throw new BusinessException("解析评分结果失败", e);
```

## 关键经验总结

### 1. 查阅官方文档的重要性
**问题：** MyBatis-Plus 3.5.9分页插件找不到类

**错误做法：** 盲目尝试不同的包路径或降级版本

**正确做法：**
1. 查阅官方文档：https://baomidou.com/plugins/pagination/
2. 发现版本变更说明
3. 按文档添加正确的依赖

### 2. 架构迁移的完整性
**问题：** Repository文件删除操作失败

**教训：**
- 文件删除需要确认文件系统状态
- PowerShell删除命令需要正确的权限和路径
- 编译前务必确认旧代码已完全清理

### 3. 异常处理的正确性
**问题：** BusinessException参数类型错误

**最佳实践：**
- 参考项目中已有的工作示例
- 异常链应该保留原始异常（传入cause参数）
- 不要创建不存在的构造函数重载

### 4. 系统化调试流程
**遵循步骤：**
1. ✅ Phase 1: Root Cause Investigation（根因分析）
2. ✅ Phase 2: Pattern Analysis（模式分析）
3. ✅ Phase 3: Hypothesis and Testing（假设测试）
4. ✅ Phase 4: Implementation（实施修复）

**避免：**
- ❌ 随机猜测
- ❌ 一次修改多处
- ❌ 跳过根因分析直接修复
- ❌ 不验证修复是否有效
